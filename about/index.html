대표 경험

세모덕에서는 인증, 권한, DB, 게시글, 외부 굿즈 검색, 관리자 기능을 연결했고 나들이즘에서는 Flutter 화면과 FastAPI 추천 API, 소셜 로그인, 원스토어 출시 흐름을 경험했습니다.

문제 해결 방식

오류가 발생하면 화면 이벤트, API 요청, 인증 상태, DB 정책을 나누어 확인합니다. 세모덕에서는 외부 상품 상세 URL 구조와 RLS 권한 불일치 문제를 실제 데이터 흐름 기준으로 고쳤습니다.

지원 방향

현재는 프론트엔드와 백엔드 흐름을 함께 이해하고, AI 기능을 사용자 화면에 붙일 수 있는 신입 개발자 포지션을 찾고 있습니다. 빠르게 구현하되 배포와 운영에서 생기는 문제까지 확인하는 개발자를 지향합니다.
