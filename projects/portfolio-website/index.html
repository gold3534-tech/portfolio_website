Overview

프로젝트 소개

프로젝트 목록, 상세 페이지, 기술 스택, 소개, GitHub 정보를 하나의 정적 사이트 안에서 탐색할 수 있도록 구성했습니다.

Problem

기획 배경

단순 원페이지보다 실제 라우트와 상세 페이지가 있는 포트폴리오가 프로젝트 흐름을 더 명확하게 보여줄 수 있습니다.

Flow

시스템 또는 데이터 흐름

공통 프로젝트 데이터 → 목록 카드 렌더링 → 상세 라우트 렌더링 → GitHub 공개 API 정보 표시

다중 페이지 정적 라우트프로젝트 필터프로젝트 상세 페이지반응형 헤더와 모바일 메뉴GitHub API fallback UISEO 파일 구성

Troubleshooting

문제 해결 경험

문제

기존 정적 사이트는 원페이지와 모달 중심이라 새로고침 가능한 상세 경로가 없었습니다.

원인

프로젝트 데이터를 화면 내부에서 직접 다루고 실제 파일 기반 라우트가 분리되어 있지 않았습니다.

해결 과정

공통 데이터 파일과 폴더 기반 index.html 라우트를 만들고 같은 렌더링 스크립트를 공유하도록 바꿨습니다.

배운 점

정적 사이트에서도 파일 구조와 데이터 분리를 잘 잡으면 포트폴리오 규모 확장이 가능합니다.
